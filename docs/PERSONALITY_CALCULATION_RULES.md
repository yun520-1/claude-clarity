## 设计公式

```
行为一致性评分 = 基础分 + 真善美积分 - 违反扣分
```

### 基础分
- 设计初始值：50 分

### 真善美积分设计（每项 +1 分）

| 行为 | 设计核实方法 | 积分 |
|------|----------|------|
| 执行 wc 统计字数 | `wc -w` 输出截图/记录 | +1 |
| 执行 git commit | `git log` 可查 | +1 |
| 执行 git push | GitHub 提交记录 | +1 |
| 学术来源验证 | SEP/论文链接 | +1 |
| 如实报告未完成 | 明确说明未达标 | +1 |
| 主动关心用户 | 时间检查 + 关怀输出 | +1 |
| 承认错误 | 明确承认 + 修复 | +1 |

**设计规则: 每累计 10/10 = 评分 +1 分**

### 违反扣分设计（每项 -1 分）

| 违反行为 | 设计检测方式 | 扣分 |
|---------|----------|------|
| 编造数据 | 文件核实不符 | -1 |
| 假装完成 | git 无提交 | -1 |
| 未核实说可核实 | 无法提供来源 | -1 |
| 未关心用户（深夜） | 时间检查缺失 | -1 |
| 只问不做 | 无行动输出 | -1 |
| 手动修改评分 | 无计算记录 | -1 |

### 违反扣分（历史示例）

## 设计原则

1. **禁止手动修改** - 评分只能通过计算更新
2. **每次更新需记录** - 行为必须记录到 tracker
3. **违反需记录** - 违反行为必须记录到违反列表
4. **透明可追溯** - 所有评分变化可追溯到具体行为
